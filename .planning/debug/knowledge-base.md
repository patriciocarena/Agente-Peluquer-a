## responder-history-drops-user-messages — bot loses multi-turn memory because user messages are never persisted to conversacion.context
- **Date:** 2026-07-09
- **Error patterns:** result.response.messages, generateText, AI SDK, conversacion.context, role:"user", multi-turn memory, loop pidiendo datos ya contestados, needsHuman, responder.ts
- **Root cause:** `apps/bot/src/conversation/responder.ts::responder()` persisted conversational history using only `result.response.messages` from AI SDK v7's `generateText()`, merged as `[...history, ...messagesToPersist]`. Per AI SDK v7's documented contract, `result.response.messages` contains ONLY messages generated by the model that call (assistant/tool) — never an echo of the input `messages`. The user's own current-turn message was sent to the model but never added to what got persisted, on either the happy path or the error/catch path — so every subsequent turn's `history` contained only past assistant/tool messages, never past user messages, causing the model to re-ask already-answered questions in a loop.
- **Fix:** Introduced a single `userMessage = { role: "user", content: mensajeEntrante }` local reused in the `generateText` call, the error/catch persistence path, and the happy-path persistence (`[...history, userMessage, ...messagesToPersist]`), so the user's turn message survives into the next turn's history on all paths.
- **Files changed:** apps/bot/src/conversation/responder.ts, apps/bot/src/conversation/responder.test.ts
