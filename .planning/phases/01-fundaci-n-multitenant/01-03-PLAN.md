<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dashboard user (browser + JWT) → Postgres | RLS is the real security boundary; a logged-in owner must never read another tenant's rows |
| bot service (service_role) → Postgres | service_role BYPASSES RLS entirely; isolation is code-enforced (plan 05), not DB-enforced |
| superadmin → all tenants | Cross-tenant access is app-gated via service_role in an isolated route, never via relaxed RLS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | tenant-scoped tables (cross-tenant read) | mitigate | RLS enabled on every tenant table with a uniform `tenant_id = auth_tenant_id()` policy; verified behaviorally with 2 tenants in plan 05 (CORE-01/02) |
| T-03-02 | Elevation of Privilege | `auth_tenant_id()` SECURITY DEFINER helper | mitigate | Locked `search_path`, STABLE, EXECUTE only to `authenticated`; reads only the caller's own perfil row via `auth.uid()` (not client-spoofable) |
| T-03-03 | Spoofing | tenant identity via JWT claim | mitigate | Tenant resolved from the DB perfil row (D-07), not a mutable/desyncable JWT claim — removes claim-forgery and stale-claim risk |
| T-03-04 | Elevation of Privilege | superadmin cross-tenant path | mitigate | No `superadmin` branch in any RLS policy (D-06); cross-tenant is service_role-gated app-side only (Phase 2), keeping the RLS surface strictly single-tenant |
| T-03-05 | Tampering | double-booking under concurrency | mitigate | DB-level `EXCLUDE USING gist ... WHERE estado != 'cancelado'` makes overlaps structurally impossible regardless of app races (CORE-05); load test formalized in Phase 7/SEC-02 |
| T-03-06 | Information Disclosure | `tenant.whatsapp_token` at rest | accept (this phase) | Column exists plaintext now; encryption (Vault/AES-GCM) explicitly deferred to Phase 7/SEC-01 (CONTEXT Deferred Ideas). Documented residual risk; no real tenant tokens seeded in v1 dev |
