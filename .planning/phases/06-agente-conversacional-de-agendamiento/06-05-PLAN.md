## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WhatsApp user message → generateText | Todo el contenido del mensaje es no confiable; el system prompt y las closures son la defensa |
| result.text (modelo) → cliente | El texto del modelo puede narrar un booking inexistente (confirmación fantasma) — no se confía sin verificar el trace |
| needsHuman flag → invocación del agente | El flag vive fuera del control del modelo (leído por inboundWorker), no lo decide el LLM |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-16 | Repudiation / Tampering | gate D-12 (confirmación fantasma) | mitigate | El envío de lenguaje de cierre se gatea en código sobre result.steps (turno_id real de confirmarTurno/reagendarTurno); sin él → mensaje seguro + needsHuman + log de incidente. Guardrail online catastrófico #1 (Section 6) |
| T-06-17 | Elevation of Privilege / Information Disclosure | prompt injection vía mensaje | mitigate | negocioId/clienteId cerrados en closure antes de generateText; ninguna tool los recibe del modelo; system prompt refuerza el rechazo — defensa ESTRUCTURAL, no solo prompt (D-13/BOT-11) |
| T-06-18 | Denial of Service | loop del modelo sin fin | mitigate | stopWhen: isStepCount(6) acota el loop; un modelo confundido no puede quemar tool calls indefinidamente (Pitfall 1) |
| T-06-19 | Tampering | handoff eludido por el modelo | mitigate | El skip por needsHuman lo ejecuta inboundWorker (fuera del control del LLM), no una decisión del modelo (D-11) |
| T-06-20 | Information Disclosure | swallow silencioso de errores de tool | mitigate | try/catch loguea NoSuchToolError/InvalidToolInputError distinto — un error de tool no se narra como éxito (Section 3 Key Abstractions) |
| T-06-SC | Tampering | npm installs | mitigate | Este plan no instala paquetes (installs en 06-02) |
