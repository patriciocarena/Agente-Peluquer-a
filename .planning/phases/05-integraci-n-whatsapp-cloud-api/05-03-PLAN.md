  <action>
    Create `apps/bot/src/whatsapp/graphClient.ts` exporting `async sendWhatsappMessage(negocioId: string, to: string, body: string)`. Build the payload `{ messaging_product: "whatsapp", recipient_type: "individual", to, type: "text", text: { preview_url: false, body } }`. Read the API version from `loadEnv().WHATSAPP_GRAPH_API_VERSION` — never hardcode a version literal at the call site. When `loadEnv().WHATSAPP_LIVE` is false, log via the module logger and return `{ messages: [{ id: \`mock.${Date.now()}\` }] }` without any network call (D-01). To keep it unit-testable without a network, accept an optional injected `deps` param defaulting to the real `{ fetch, getWhatsappToken, getPhoneNumberId, log }` (mirror booking.ts's optional-deps style) so the test can pass a fetch spy. Create `apps/bot/src/whatsapp/graphClient.test.ts` covering the four behaviors: assert no fetch call when live=false; assert fetch called with the correct URL/method/Authorization when live=true (spy); assert throw on non-ok; assert throw/log on 200-with-error-body. Toggle `WHATSAPP_LIVE` via `process.env` around cases and restore.
