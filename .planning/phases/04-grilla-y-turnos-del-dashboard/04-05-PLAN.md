<objective>
UI de alta manual, reagendado y detalle de turno (APPT-03/04/05/06), consumiendo las Server Actions de Plan 04 (turnos + slots) y de Plan 03 (clientes). Cuatro componentes cliente que se componen entre sí:

1. `cliente-search.tsx` (D-09): input de búsqueda por teléfono → lista de resultados; si vacío, formulario inline (teléfono pre-cargado + nombre opcional) que crea el cliente al vuelo. Devuelve el `clienteId` seleccionado al padre.
2. `slot-selector.tsx` (D-10/D-13, compartido): selector de profesional elegible (`profesionalesElegibles`) + grid de chips de horarios reales (`obtenerSlotsDisponibles`). Devuelve `{ profesionalId, inicio, fin }`.
3. `turno-form-dialog.tsx` (alta manual + reagendar, dual-mode): compone cliente-search + slot-selector; en alta llama `crearTurnoManual`, en reagendar llama `reagendarTurno`.
4. `turno-detail-sheet.tsx` (D-04, APPT-03): `Sheet` lateral con cliente, servicios+precio, total, horario, profesional; acciones "Reagendar" (abre turno-form-dialog en modo reagendar) y "Cancelar turno" (confirmación simple D-12 → `cancelarTurno`).

Purpose: Completar el flujo operativo de turnos del dueño en piezas que la grilla (Plan 07) monta.
Output: cuatro componentes cliente.
</objective>
